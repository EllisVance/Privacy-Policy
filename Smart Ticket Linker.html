Privacy Policy

Smart Ticket Linker · Effective date: May 19, 2026

Short version: Smart Ticket Linker does not collect, sell, transmit, or remotely store personal data. Ticket parsing and clipboard formatting happen locally in your browser. The extension only stores your own settings, such as enabled state, Markdown template, and clipboard mode.

1. Overview

Smart Ticket Linker is a Chrome extension that helps users copy the current Jira, Linear, GitHub, or web page as a clean rich link with a Markdown fallback. When you press the configured shortcut, the extension reads limited information from the current active page, formats it locally, and writes it to your clipboard.

This Privacy Policy explains what data the extension processes, how that data is used, and what is not collected.

2. Data We Do Not Collect

Smart Ticket Linker does not collect, transmit, sell, share, or remotely store:

  • Personally identifiable information
  • Browsing history
  • Authentication credentials
  • Cookies
  • Analytics or tracking identifiers
  • Clipboard contents
  • Ticket content beyond the limited page metadata needed for the copy action

3. Data Processed Locally

After a direct user action, such as pressing the copy shortcut, the extension may process the following information from the current active tab:

  • The current page URL
  • The current page title
  • Visible ticket metadata, such as Jira issue key, Linear issue ID, GitHub issue number, pull request number, and visible issue title

This information is processed locally in your browser only to create the formatted clipboard output. It is not sent to any external server.

4. Clipboard Use

The extension uses the clipboard only for its core feature: writing a formatted ticket or page link after you invoke the copy command. Depending on your settings, the extension may write:

  • text/html for rich text paste targets such as Slack, Notion, Discord, and document editors
  • text/plain for Markdown fallback in plain text editors and code editors

Smart Ticket Linker does not read your clipboard contents.

5. Stored Settings

The extension uses chrome.storage.sync to store user preferences, including:

  • Whether the extension is enabled
  • Your Markdown template
  • Your selected clipboard mode

These settings are stored by Chrome so your configuration can persist across browser sessions. The extension does not store page contents, ticket descriptions, browsing history, or clipboard history.

6. Permissions

activeTab

Used to access the current active tab after a direct user action, such as pressing the copy shortcut.

scripting

Used to inject the local content script into the current active page when needed for parsing and clipboard writing.

clipboardWrite

Used to write the formatted rich link or Markdown link to the clipboard after user activation.

storage

Used to save user preferences such as enabled state, Markdown template, and clipboard mode.

Host access

Used to support Jira, Linear, GitHub, and general web-page fallback behavior. The extension only processes page metadata locally when you invoke the copy command.

7. Third-Party Services

Smart Ticket Linker does not use external analytics, advertising SDKs, tracking services, remote APIs, or third-party data processors.

8. Data Sharing

Smart Ticket Linker does not sell, rent, share, or transfer user data to third parties.

9. Security

The extension is designed to minimize data handling. All parsing and formatting happens locally in the browser. No remote server is used for ticket extraction or clipboard formatting.

10. Changes to This Policy

This Privacy Policy may be updated if the extension's functionality or data handling changes. The effective date at the top of this page will be updated when material changes are made.

11. Contact

For privacy questions, please contact the publisher through the Smart Ticket Linker listing on the Chrome Web Store.

1. 插件用途

Smart Ticket Linker 是一个 Chrome 插件，用于将当前 Jira、Linear、GitHub 或普通网页一键复制为干净的富文本链接，并提供 Markdown 纯文本降级。当你按下配置好的快捷键时，插件会读取当前活动页面的有限信息，在本地格式化后写入剪贴板。

2. 不收集的数据

插件不会收集、传输、出售、共享或远程存储个人身份信息、浏览历史、登录凭据、Cookie、分析标识符、剪贴板内容或完整任务内容。

3. 本地处理的数据

在你主动触发复制操作后，插件可能会在本地处理当前页面 URL、页面标题，以及可见的任务元数据，例如 Jira issue key、Linear issue ID、GitHub issue number、Pull Request number 和可见标题。这些信息只用于生成剪贴板内容，不会发送到外部服务器。

插件只在你触发复制命令后写入剪贴板。根据设置，插件可能写入 text/html 富文本链接和 text/plain Markdown 文本。插件不会读取你的剪贴板内容。

5. 设置存储

插件使用 chrome.storage.sync 保存启用状态、Markdown 模板和剪贴板模式。插件不会保存网页内容、任务描述、浏览历史或剪贴板历史。

6. 权限用途

activeTab 用于在用户主动触发复制后访问当前活动标签页；scripting 用于按需注入本地内容脚本；clipboardWrite 用于写入格式化后的链接；storage 用于保存用户设置；网站访问权限用于支持 Jira、Linear、GitHub 和普通网页 fallback。

7. 第三方服务和数据共享

插件不使用外部分析、广告 SDK、追踪服务、远程 API 或第三方数据处理器，也不会向第三方出售、出租、共享或转移用户数据。
